# week-task-manager

週間スケジュール管理アプリ。曜日ごとに予定・タスクを時間軸で管理するカレンダー的なフロントエンドアプリ。

## 技術スタック

- **React** — UIコンポーネント
- **Vite** — 開発サーバー・ビルドツール
- **JavaScript (ES6+)** — ロジック

## ファイル構成

```
week-task-manager/
├── index.html         # エントリポイント
├── package.json
├── vite.config.js
├── src/
│   ├── main.jsx        # アプリのマウント
│   ├── App.jsx          # ルートコンポーネント
│   ├── components/      # UIコンポーネント
│   ├── hooks/           # カスタムフック
│   └── data/            # 初期データ・型定義
└── public/
```

## 開発ルール

- コンポーネントは関数コンポーネント + Hooksで記述する。クラスコンポーネントは使わない。
- `dangerouslySetInnerHTML` の使用は禁止。XSS防止のため通常のJSX描画を使うこと。
- 状態管理はまずReact標準のuseState/useContextで行う。状態が複雑化するまで外部ライブラリ（Redux等）は導入しない。
- `console.log` はデバッグ用途のみ。本番コードに残さない。
- コメントは「なぜ（WHY）」が自明でない箇所にのみ記述する。
- スタイルはコンポーネント単位のCSSファイル（`ComponentName.css`）で管理し、インラインstyleは避ける。

## アプリ仕様

- 表示単位：1週間（月〜日、または日〜土）
- 曜日ごとに予定・タスクを登録・編集・削除できる
- タスクには時間帯・タイトル・完了状態を設定できる
- 週の切り替え（前週・次週への移動）ができる
- タスクデータはブラウザのlocalStorageに保存し、再読み込み後も保持する
- 働く曜日・時間（開始/終了）を設定でき、後から再設定もできる（未設定時は設定フォームのみ表示）
- 設定完了後にタスク一覧と1週間カレンダーを表示する
- タスクは「毎日タスク」「週1タスク」「単発タスク」の3種類に分けて登録できる
  - 毎日タスク：すべての日に表示。特定の時刻は持たない（作業時間のみ）
  - 週1タスク：指定した曜日にのみ表示。特定の時刻は持たない（作業時間のみ）
  - 単発タスク：締切の日時を設定し、「カレンダーに設定」ボタンで自動的に空き時間（時刻つき）へ登録する。カレンダーから直接追加することはできない（登録はタスク一覧からのみ行う）
- 完了状態はタスクの出現日（日付）ごとに個別に管理する（繰り返しタスクも日ごとにチェックできる）
- タスクには作業時間（所要時間）を設定できる。カレンダーへの自動登録時は、働く曜日・時間の範囲内で他のタスク（毎日/週1タスクの分は時間帯を問わずその日の枠から差し引く）と重複しない空き時間を探して登録する
  - 1日の空き時間で作業時間が収まらない単発タスクは、複数の日に分割して登録する（締切までに収まらない分は未登録のまま残る）
- タスクの出現ごとに進捗（10%刻みのプログレスバー）と完了チェックボックスを設定できる。完了チェックは進捗100%と連動する
- カレンダーには各タスクの作業時間を表示する
- 進捗率と残りの作業時間から達成度を判定し、カレンダーに「順調」「遅延」「前倒し」のいずれかを表示する（完了済み・残り作業時間0のタスクには表示しない）
  - 残り作業時間 = 作業時間 ×（1 − 進捗率）
  - 利用可能時間：過去の日付は0、未来の日付は作業時間そのまま、当日は「終業時刻 − 現在時刻」
  - 達成率 = 利用可能時間 ÷ 残り作業時間 × 100（90%未満は遅延、90〜110%は順調、110%超は前倒し）
- 登録済みタスクは「タスクを編集」フォームから内容を変更できる（種別の変更は不可。単発タスクは編集するとカレンダー登録がリセットされ、再度「カレンダーに設定」が必要になる）
- 登録済みタスク一覧は「毎日タスク」「週1タスク」「単発タスク」ごとに区切って表示する
- 「カレンダーに設定」ボタンは単発タスクのグループにのみ表示する
- レイアウトはタスク一覧を左、1週間カレンダーを右に表示する（画面が狭い場合は縦に並べる）
- カレンダーは曜日（日付）を縦に並べて表示する。各日には働く時間内の残り時間（空き時間）を表示する
- カレンダーへの直接のタスク追加フォームは持たない（登録はタスク一覧からのみ）
- タスクのタイトルは省略せず、1行に収まらない場合は複数行で折り返して表示する（削除ボタンが隠れないようにする）
- タスク一覧画面は「タスクを登録」（種別・曜日はラジオボタン、締切は項目名つき）と「登録済みタスク」を枠で区切って表示する
- デザインはビビッドな配色（紫×ピンクのアクセント）と角丸・シャドウを基調にしたポップなトーンにする

## 動作確認

```
npm install
npm run dev
```

## よくある作業パターン

- **コンポーネント追加**: `src/components/` に `.jsx` と対応する `.css` を追加する
- **データ構造変更**: `src/data/` 内の初期データ・型定義を編集する
- **状態ロジック変更**: `src/hooks/` のカスタムフックを編集する

## Git運用ルール

**コードを変更するたびに、必ず GitHub にプッシュすること。**

```bash
git add <変更ファイル>
git commit -m "変更内容を簡潔に記述"
git push origin main
```

- コミットメッセージは日本語可。変更の「なぜ」が伝わる内容にする。
- `git add .` や `git add -A` は機密ファイル（.env 等）を誤って含めるリスクがあるため、原則としてファイルを個別に指定する。
- force push（`--force`）は禁止。main ブランチへの force push は特に厳禁。
- `.env` などの機密情報を含むファイルはコミットしない。`.gitignore` に必ず追加する。
