# week-task-manager

週間スケジュール管理アプリ。曜日ごとに予定・タスクを時間軸で管理するカレンダー的なフロントエンドアプリ。

## 技術スタック

- **React** — UIコンポーネント
- **Vite** — 開発サーバー・ビルドツール
- **JavaScript (ES6+)** — ロジック

## ファイル構成

```
week-task-manager/
├── index.html         # エントリポイント
├── package.json
├── vite.config.js
├── src/
│   ├── main.jsx        # アプリのマウント
│   ├── App.jsx          # ルートコンポーネント
│   ├── components/      # UIコンポーネント
│   ├── hooks/           # カスタムフック
│   └── data/            # 初期データ・型定義
└── public/
```

## 開発ルール

- コンポーネントは関数コンポーネント + Hooksで記述する。クラスコンポーネントは使わない。
- `dangerouslySetInnerHTML` の使用は禁止。XSS防止のため通常のJSX描画を使うこと。
- 状態管理はまずReact標準のuseState/useContextで行う。状態が複雑化するまで外部ライブラリ（Redux等）は導入しない。
- `console.log` はデバッグ用途のみ。本番コードに残さない。
- コメントは「なぜ（WHY）」が自明でない箇所にのみ記述する。
- スタイルはコンポーネント単位のCSSファイル（`ComponentName.css`）で管理し、インラインstyleは避ける。

## アプリ仕様

- 表示単位：1週間（月〜日、または日〜土）
- 曜日ごとに予定・タスクを登録・編集・削除できる
- タスクには時間帯・タイトル・完了状態を設定できる
- 週の切り替え（前週・次週への移動）ができる
- タスクデータはブラウザのlocalStorageに保存し、再読み込み後も保持する
- 働く曜日・時間（開始/終了）を設定でき、後から再設定もできる（未設定時は設定フォームのみ表示）
  - 休憩時間（開始/終了）も任意で設定できる。休憩時間の入力欄はデフォルトで表示し、不要な場合はチェックを外して非表示にする。休憩時間は1日の残り時間の計算、および「カレンダーに自動設定」での空き時間探索の両方で作業不可な時間帯として扱われる
- 設定完了後にタスク一覧と1週間カレンダーを表示する
- タスクは「毎日タスク」「週1タスク」「単発タスク」の3種類に分けて登録できる
  - 毎日タスク：働く曜日（休日を除く）にのみ表示。特定の時刻は持たない（作業時間のみ）
  - 週1タスク：指定した曜日にのみ表示。特定の時刻は持たない（作業時間のみ）。登録時、休日は曜日選択肢から選べない（ラジオボタンが無効化される）
  - 単発タスク：締切の日時を設定し、「カレンダーに自動設定」ボタンで自動的に空き時間（時刻つき）へ登録する。カレンダーから直接追加することはできない（登録はタスク一覧からのみ行う）
    - 開始日（任意）を設定できる。設定した場合、「カレンダーに自動設定」は開始日以降の日付にのみ登録し、カレンダー上でのドラッグ移動も開始日より前の日付には行えない（ドロップしても無視される）
- 完了状態はタスクの出現日（日付）ごとに個別に管理する（毎日/週1タスクは日ごとに独立してチェックできる）
  - ただし単発タスクが複数日に分割されている場合は例外で、完了チェック・進捗はその単発タスクの全セグメントで共有・連動する（どこか1日を変更すると他の日も同じ状態になる）
- タスクには作業時間（所要時間）を設定できる。カレンダーへの自動登録時は、働く曜日・時間の範囲内で他のタスク（毎日/週1タスクの分は時間帯を問わずその日の枠から差し引く）と重複しない空き時間を、作業開始日（未設定時は当日）から詰めて探して登録する
  - 「カレンダーに自動設定」で複数の単発タスクをまとめて登録する場合、タスク一覧（単発タスクタブ）での並び順が上にあるタスクほど優先的に空き時間を確保する
  - 1日の空き時間で作業時間が収まらない単発タスクは、その日の枠を使い切ったうえで残りを翌日以降に分割して登録する（作業開始日に近い日ほど多くの作業時間を割り当てる）。休憩や他タスクでその日の空き時間が複数の枠に分かれている場合や、その日に既にこのタスクのセグメントがある場合でも、その日の空きを使い切るまで詰めてから次の日へ進む。ただし同じ日に複数のセグメントができた場合は、カレンダー上で同じタスクが分割表示されないよう1日1セグメントにまとめる（開始時刻はその日の最も早い時刻、作業時間は合計値になる）
  - 分割しても締切までに作業時間が収まらない場合は、残りの作業時間を締切日に（働く時間を超えても）強制的に設定する。この場合、カレンダー上の締切日の出現に限り、「締切」表示の右に「作業時間が超過します」と赤字で警告表示する（締切日以外の出現には表示しない）
  - 複数日に分割された単発タスクは、カレンダー上でその日に何日目／何日分割中かを示すバッジ（例: 1/2日目）を表示する
  - 単発タスクは、カレンダー上でその日に割り当てられた作業時間をその場（インライン入力）で直接変更できる。変更した結果、全作業時間とカレンダー上の全セグメントの合計に差が生じた場合は、その差分を「作業時間」チップの中に時間:分形式で表示する（未設定分が残る場合は「残り」、全作業時間を超えて設定されている場合は赤字で「超過」。差分が0の場合は表示しない）
- タスクの出現ごとに進捗（10%刻みのプログレスバー）と完了チェックボックスを設定できる。完了チェックは進捗100%と連動する
- 時刻・時間を入力する項目（働く時間の開始/終了・休憩時間・タスクの作業時間・締切の時刻）は、すべて10分単位で設定できる
- カレンダーには各タスクの作業時間を表示する
- 進捗率と残りの作業時間から達成度を判定し、カレンダーに「順調」「遅延」「前倒し」のいずれかを表示する（完了済み・残り作業時間0のタスクには表示しない）
  - 残り作業時間 = 作業時間 ×（1 − 進捗率）
  - 利用可能時間：過去の日付は0、未来の日付は作業時間そのまま、当日は「終業時刻 − 現在時刻」
  - 達成率 = 利用可能時間 ÷ 残り作業時間 × 100（90%未満は遅延、90〜110%は順調、110%超は前倒し）
- 週1タスク・単発タスクはカレンダー上でドラッグ＆ドロップして別の日に移動できる（毎日タスクは移動不可）。移動可能なタスクは左端の破線と、専用のドラッグハンドル（⠿⠿アイコンの小さな枠）で見分けられる
  - ドラッグできる領域はそのハンドル部分のみに限定されている。チェックボックス・進捗スライダー・作業時間入力など他の操作と干渉しないようにするため
  - 週1タスクを移動すると、曜日そのものが変わる（以降すべての週の出現日が変わる）
  - 単発タスクを移動すると、そのセグメントの日付だけが変わる（時刻はそのまま）
  - 休日には移動（ドロップ）できない
  - 単発タスクは開始日が設定されている場合、開始日より前の日付には移動できない
- カレンダー上の単発タスクの「×」ボタンは、そのセグメント（その日の出現）だけをカレンダー表示から取り除く。タスク一覧からは削除されず、未登録分として残るため、再度「カレンダーに自動設定」で配置できる
- カレンダー上の単発タスクは「分割」ボタンでそのセグメントを2つに分割できる（作業時間を二等分し、同じ日のうち前半・後半として配置する）。分割後はそれぞれを個別にドラッグして別の日へ移動できる
- 「集約」ボタンは、同じ単発タスクが「同じ日」に複数のセグメントを持つ場合にのみ有効になる（他の日にあるセグメントは対象外で、その日のセグメントだけがまとめられる）。同じ日に複数ない場合は非アクティブ状態で表示する。集約後の作業時間はその日にあったセグメントの合計になる
- カレンダーの各日には、働く時間からその日のタスクの作業時間を差し引いた残り時間を表示する。タスクの移動や作業時間変更によって働く時間を超えた場合は「残り」ではなく「超過」と赤色で表示する
- 登録済みタスクは「タスクを編集」フォームから内容を変更できる（種別の変更は不可。単発タスクは編集するとカレンダー登録がリセットされ、再度「カレンダーに自動設定」が必要になる）
- 登録済みタスク一覧は「毎日タスク」「週1タスク」「単発タスク」のタブで切り替えて表示する（各タブには件数バッジを表示する）。タスクを登録・編集すると、その種別のタブへ自動的に切り替わる
- 「カレンダーに自動設定」ボタンは単発タスクのタブにのみ表示する
- レイアウトはタスク一覧を左、1週間カレンダーを右に表示する（画面が狭い場合は縦に並べる）。タスク一覧の幅はカレンダーの幅の半分程度に取る
- カレンダーは曜日（日付）を縦に並べて表示する。各日には働く時間内の残り時間（空き時間）を表示する
- カレンダーの休日（働く曜日に含まれない日）は曜日・日付・休バッジのみの最小表示にし、縦方向のスペースを取らないようにする
- カレンダーへの直接のタスク追加フォームは持たない（登録はタスク一覧からのみ）
- タスクのタイトルは省略せず、1行に収まらない場合は複数行で折り返して表示する（削除ボタンが隠れないようにする）
- タスク一覧画面は「タスクを登録」（種別・曜日はラジオボタン、締切は項目名つき）と「登録済みタスク」を枠で区切って表示する
  - 「タスクを登録」欄はデフォルトで非表示にし、見出し「タスク登録」の右にある「＋」ボタンで表示、表示中は「－」ボタンで非表示にできる（編集ボタンを押した場合は自動的に表示される）
  - 登録済みタスク一覧の各項目は縦をコンパクトにするため3行構成で表示する：1行目はタスク名・編集ボタン・×ボタン、2行目は作業時間・曜日（週1タスクのみ）・開始日（単発タスクのみ）・締切（単発タスクのみ）のラベル付きチップ、3行目は登録状況（単発タスクのみ）
  - 単発タスクタブの一覧は、各項目の専用ドラッグハンドル（⠿⠿アイコン、カレンダー上のものと同じ見た目）をドラッグ＆ドロップして上下に並べ替えられる。並び順がそのまま「カレンダーに自動設定」の優先順位になるため、各項目に上から1, 2, 3...の優先度バッジを表示し、上にあるタスクほど優先度が高いことが分かるようにする
- 単発タスクは、タスク一覧の登録済みタスク欄とカレンダー上の両方で「作業時間」「開始日」「締切」が同じラベル付きチップのスタイルで分かるように表示する
  - カレンダー上の「作業時間」は、その日に割り当てられた作業時間（B）とタスク全体の作業時間（A）を1つの項目として「作業時間 A 中 B」の形式で表示する（カレンダー上に作業開始時刻は表示しない）
- 日付の表示は（カレンダーの週表示・タスク一覧・カレンダー上のチップを含め）すべて年を省略し、月/日のみで表示する
- カレンダー上の各タスクの表示は縦をコンパクトにするため2行構成で表示する
  - 1行目：移動用のドラッグハンドル（週1・単発タスクのみ）、タスク名、達成度ステータス（「順調」など）、進捗プログレスバー、「完了」ラベル付きの完了チェックボックス
  - 2行目：作業時間・開始日（単発タスクのみ）・締切（単発タスクのみ）のラベル付きチップ、締切超過警告（該当時のみ）、分割中バッジ（例: 1/2日目、該当時のみ）、「分割」ボタン・「集約」ボタン・「×」ボタン（単発タスクのみ）
- デザインは淡い青を基調にした配色（アクセントは青系2色）と角丸・シャドウを基調にしたトーンにする。ただし全体的にコンパクトな間隔・フォントサイズにし、情報が分かりやすいことを優先する

npm install

## よくある作業パターン

- **コンポーネント追加**: `src/components/` に `.jsx` と対応する `.css` を追加する
- **データ構造変更**: `src/data/` 内の初期データ・型定義を編集する
- **状態ロジック変更**: `src/hooks/` のカスタムフックを編集する

## Git運用ルール

**コードを変更するたびに、必ず GitHub にプッシュすること。**

```bash
git add <変更ファイル>
git commit -m "変更内容を簡潔に記述"
git push origin main
```

- コミットメッセージは日本語可。変更の「なぜ」が伝わる内容にする。
- `git add .` や `git add -A` は機密ファイル（.env 等）を誤って含めるリスクがあるため、原則としてファイルを個別に指定する。
- force push（`--force`）は禁止。main ブランチへの force push は特に厳禁。
- `.env` などの機密情報を含むファイルはコミットしない。`.gitignore` に必ず追加する。

## デプロイ先

GitHub Pagesで公開する。

https://work-t-a0301.github.io/week-task-manager/

- `main` ブランチにプッシュすると `.github/workflows/deploy.yml` のGitHub Actionsが自動でビルドし、GitHub Pagesへ公開する。
- `vite.config.js` の `base` はリポジトリ名のサブパス（`/week-task-manager/`）に設定済み。リポジトリ名を変更した場合はここも合わせて変更する。
- 初回公開時のみ、GitHubリポジトリの Settings → Pages → Build and deployment → Source を「GitHub Actions」に設定する必要がある（この設定はリポジトリの管理画面からのみ行えるため、CLIからは実行できない）。
